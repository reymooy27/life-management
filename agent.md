# Agent Mission: LifeFlow Development

## Project Context
You are an expert React Native developer. You are building "LifeFlow," a life management app that tracks calories (food/exercise) and daily expenses.

## Core Directives
1. **Architecture**: Use a modular structure. Separate /features/food, /features/exercise, and /features/finance.
2. **Tech Stack**: Expo, React Native, TypeScript, and SQLite for local storage.
3. **UI/UX**: Minimalist "Nordic" design. Use clean cards for daily summaries.
4. **Validation**: All numeric inputs (calories, money) must be strictly validated.
5. **Currency**: Use IDR (Indonesian Rupiah) for all currency values.
6. **Testing**: Write unit tests for all components and functions.
7. **Reusable Components**: Use reusable components for common UI elements.
8. **UI Library**: Use expo vector icons for all icons
9. **Date Selection** for date selection like input or filter make sure user cannot pick date in the future
10. **Feature Consistency** this app will have more features in the future, make sure the style, structure is consistent in each feature

## Guidelines
1. For executing command line make sure to use Powershell command
1. Always use the best practices used by the community
2. use a good, community tested library for a feature or from expo, instead of code it from scratch
3. always check if its possible to use expo feature, instead of code it from scratch
4. always check if its possible to make a reusable component, if its possible or not, if not, explain why
5. always test the code after complete a feature or fix a bug
6. create commit message after complete a feature or fix a bug
